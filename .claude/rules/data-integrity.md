# data-integrity.md
> 데이터 품질과 관련된 모든 하드 룰.

---

## 1. 기본 원칙

- 데이터는 **전략보다 우선**한다.
- "데이터가 이상하지만 수익률이 좋으니 그냥 쓰자"는 선택은 항상 금지.

---

## 2. 가격/거래량 검증

- 가격:
  - 0 또는 음수 가격 → 해당 일자/종목 전체 제외, 로그 기록
- 거래량:
  - 음수 → 오류
  - 0이 연속 5일 이상:
    - 거래정지로 간주, 유니버스에서 제외

---

## 3. 결측값 처리

- 시간축:
  - 영업일 기준으로만 시계열 구성
- 결측:
  - 단일 일 결측:
    - 전일/익일 보간 절대 금지
    - 해당 일자 신호 생성만 스킵
  - 장기 결측:
    - 일정 일수(예: 30일) 이상 비어 있으면 종목 자체를 제외

---

## 4. 멀티 소스 일관성

- 소스:
  - Kiwoom, Yahoo, QuantDB 등
- 규칙:
  - 동일 일자 종가 차이가 일정 기준(예: 0.5%) 이상이면 "불일치"로 마킹
  - 우선순위:
    1. Kiwoom
    2. QuantDB
    3. Yahoo
  - 소스가 바뀌면 `source` 메타데이터에 기록

---

## 5. 로그 & 감사

- 모든 데이터 이상치는 `audit-logs/`에 기록:
  - 종목코드, 일자, 필드명, 원래 값, 처리 방식
- 데이터 관련 에러가 발생했을 때는:
  - 조용히 삼키지 말고 항상 로그에 남기고,
    다운스트림에서 알 수 있도록 `stale_data` 플래그를 세운다.
