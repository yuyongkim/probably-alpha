# frontend-ui.md
> 프론트엔드(UI) 구현 시 반드시 지켜야 할 룰.

---

## 1. UX 원칙

- 이 대시보드는 "트레이더용 운영판"이다:
  - 정보는 **의사결정 순서**대로 배치
  - 차트/그래프는 "예쁘게"보다 "결정에 도움이 되는지"가 기준

---

## 2. 정보 구조

1. 상단: 오늘 요약
   - 날짜, Top 섹터, Top 리더, 마지막 백테스트 결과 하이라이트
2. 중단: 주도 섹터/주도주 상세
   - 좌: 섹터 리스트 + 점수
   - 우: 선택된 섹터의 리더 리스트/차트
3. 하단: 백테스트/성과
   - 성과 요약, 자산 곡선, 주요 지표

---

## 3. UI 컴포넌트 규칙

- 표(Table):
  - 정렬 가능
  - 숫자 컬럼은 오른쪽 정렬
- 색상:
  - 상승/강세: 녹색 계열
  - 하락/약세: 붉은색 계열
  - 중립/참고: 회색/파란색
- 반응형:
  - 모바일에서는 한 화면에 하나의 정보 블록만 (요약 → 목록 → 상세)

---

## 4. API 연동 규칙

- 모든 API 요청은 실패 시:
  - 사용자에게 "데이터를 불러오지 못했습니다" 메시지 + 재시도 버튼
- 로딩 상태:
  - 스켈레톤 UI or 로딩 스피너 필수
- 캐시:
  - 같은 날짜의 데이터는 **새로고침 전까지** 프론트 내 메모리에서 재사용

---

## 5. 설명 가능성

- 리더 목록에서 각 종목 옆에 "왜 이 종목인가?"를 짧게 표시:
  - 예: "섹터 Top1 + TT 8/8 + VCP 수축 + EPS YoY 30%"
- 사용자가 하나의 종목을 클릭하면,
  - 차트 + 핵심 지표 + 선정 사유가 한 화면에 나오게 한다.
