# backtest.md
> 백테스트 엔진이 반드시 지켜야 할 운영 규칙.
> 수익률이 아무리 좋아도 이 문서를 어기면 결과는 폐기한다.

---

## 1. 데이터 사용 규칙

- 타임 스탬프:
  - 모든 가격/지표는 **일봉 종가 기준**으로 정의한다.
- 신호 생성:
  - 당일 종가까지의 정보만으로 신호 생성
  - 체결은 기본 `next-open`
- 실적/공시:
  - 공시일 기준 +1 거래일부터 사용

---

## 2. 거래 비용 반영

- 매수 수수료: 0.015%
- 매도 수수료: 0.015%
- 슬리피지: 기본 0.1% (시총과 유동성에 따라 가중치)
- 매도 시 거래세: 0.18%

규칙:
- **항상 비용을 반영한 결과만 보고한다.**
- "비용 없이 테스트한 결과"는 내부 참고용만, 외부 공유 금지.

---

## 3. 포지션 관리

- 최대 보유 종목 수:
  - 기본 10개 (설정값으로 조정 가능)
- 섹터 편중 제한:
  - 한 섹터 최대 3개까지 (설정 가능)
- 신호 충돌:
  - 같은 종목 중복 진입 금지
  - 기존 포지션이 있을 때 새로운 신호가 나오면:
    - 리엔트리 전략이 아니면 무시

---

## 4. 리밸런싱 및 리포트

- 리밸런싱:
  - 기본 주 1회 (금요일 종가 기준)
- 리포트:
  - 최소 월별 성과 요약, 연간 성과 요약
  - 벤치마크(KOSPI, KOSDAQ)와 비교 시 항상 같은 기간/같은 기준

---

## 5. 검증 절차

백테스트를 공식 결과로 인정하려면:

1. 코드 리뷰 1회 이상
2. 샘플 구간(예: 6개월)에서 수동 검증:
   - 실제 차트와 시그널이 맞는지
3. IS/OOS 분리 검증
4. 매개변수 변경 내역 기록 (언제, 왜 바꿨는지)
