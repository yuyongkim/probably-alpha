# BACKTEST_UI_SPEC.md
> 개인용 SEPA 백테스트 워크벤치 UI 설계.
> "전략 설정 → 실행 → 결과 해석 → 저장/비교" 흐름에 초점.

---

## 1. 메인 플로우

사용자가 하고 싶은 일은 크게 4가지다.

1. 전략을 만든다 (혹은 수정한다).
2. 기간/파라미터를 정하고 백테스트를 실행한다.
3. 결과를 보고, "왜 이런 성과인지"를 이해한다.
4. 괜찮으면 전략/결과를 저장하고, 나중에 다시 불러와 비교한다.

UI는 이 4단계를 기준으로 설계한다.

---

## 2. 화면 구조

### 2.1 상단 탭 구조

- `Strategies` 탭
  - 전략 목록, 생성/수정/복제
- `Backtests` 탭
  - 실행된 백테스트 run 목록, 결과 요약
- `Workspace` 탭
  - 현재 선택한 전략 + 선택한 run을 한 화면에서 보는 워크스페이스

---

## 3. Strategies 화면

목표: 전략 생성/수정을 쉽게 하고, 각 전략이 어떤 룰을 쓰는지 한눈에 보이게 한다.

구성:

1. 전략 리스트 (좌측)
   - 전략 이름
   - 유형(예: SEPA-Leader, Value, Short 등)
   - 최근 수정일
   - 주요 파라미터 요약 (유니버스, 리밸런싱, 손절 규칙 등)

2. 전략 편집 패널 (우측)
   - 전략 메타 정보:
     - 이름, 설명, 전략군(성장/추세/가치 등)
   - 규칙/파라미터:
     - 유니버스 필터 (시장, 최소 시총, 최소 거래대금 등)
     - 매수 조건(Alpha/Beta/Gamma/Delta 기반)
     - 매도 조건(손절/추세 이탈/리밸런싱)
     - 리밸런싱 빈도
     - 비용 설정(전략별 override가 필요하면)

3. 액션
   - New Strategy
   - Duplicate Strategy
   - Delete (soft delete)

---

## 4. Backtests 화면

목표: 각 run 별 성과를 테이블로 정리하고, 빠르게 비교할 수 있게 한다.

구성:

- Backtest run 리스트
  - 컬럼:
    - run_id
    - 전략 이름
    - 기간(start~end)
    - CAGR
    - Max Drawdown
    - Sharpe
    - 실행일/실행 시간
  - 정렬/필터:
    - 전략별 필터
    - 날짜/성과 기준 정렬

- 선택 시:
  - 우측 패널에 요약 카드:
    - "전략 X, 3년, CAGR 18.3%, MDD -22.5%, Sharpe 1.1"

- 액션:
  - View in Workspace (워크스페이스에서 상세 보기)
  - Export (CSV/JSON)
  - Re-run (같은 전략/파라미터로 재실행)

---

## 5. Workspace 화면

워크스페이스는 "현 전략 + 선택 run"을 한 화면에서 집중적으로 보는 곳이다.

### 5.1 레이아웃

1. 상단: 전략 요약
   - 전략 이름/설명
   - 핵심 룰 요약
   - 사용 파라미터(유니버스, 리밸런싱, 비용 등)

2. 중단: 성과 요약 & 자산 곡선
   - KPI 카드:
     - CAGR, MDD, Sharpe, Win Rate, Trades 수
   - 자산 곡선 차트:
     - 전략 vs 벤치마크
   - Drawdown 차트:
     - 날짜별 최대 낙폭

3. 하단 탭:

   - `Trades` 탭:
     - 거래 로그 테이블
       - 날짜, 심볼, 방향, 수량, 진입/청산가, P&L, 보유기간, 이유
   - `Positions` 탭:
     - 날짜별 포지션 스냅샷, 보유 비중, 섹터 분포
   - `Monthly` 탭:
     - 월별 수익률 heatmap/테이블
   - `Diagnostics` 탭:
     - 리스크/바이어스 체크 결과
     - 탑 드로다운 구간 리스트
     - 과최적화 의심 포인트

---

## 6. 사용자 인터랙션 플로우

### 6.1 새로운 전략 테스트

1. Strategies에서 `New Strategy` -> 규칙/파라미터 정의
2. "Run Backtest" 버튼 클릭 -> 기간/기본 파라미터 확인 -> 실행
3. Backtests 탭에 run이 생성되고, 상태는 QUEUED/RUNNING/COMPLETED
4. 완료 후 `View in Workspace`로 이동

### 6.2 기존 전략 튜닝

1. 기존 전략을 복제(Duplicate) -> 이름에 버전 표기
2. 일부 필터/파라미터를 변경
3. 동일 기간으로 백테스트 실행
4. Backtests 탭에서 여러 run 선택 후 KPI 비교 (표/간단 차트)

---

## 7. 에러/로딩/Empty 상태

- 백테스트 실행 중:
  - 진행 상태 바 + 예상 완료 시간(rough estimate)
- 데이터 부족/에러:
  - Run 상태를 FAILED로 표시하고, 사유(예: 데이터 결측, 설정 오류)를 명시
- 결과 없음:
  - 특정 전략에 백테스트가 하나도 없을 경우,
    - "아직 실행된 백테스트가 없습니다. 바로 하나 실행해 보세요." 메시지 + CTA
