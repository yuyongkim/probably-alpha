## 한 줄 요약

> **"데이터에서 패턴을 발견하되, 과적합을 철저히 경계하라. 진정한 엣지는 다른 사람들이 보지 못하는 데이터 관계에 있다."**

## 핵심 투자 철학 (5 Principles)

1. **데이터가 답이다 (Data Is the Answer)**
   - 인간의 직관이나 경제 이론보다 데이터에서 발견된 통계적 패턴을 신뢰한다.
   - 방대한 데이터셋에서 반복적으로 나타나는 비효율을 체계적으로 포착한다.

2. **과적합은 최대의 적이다 (Overfitting Is the Greatest Enemy)**
   - 백테스트에서 완벽해 보이는 모델이 실전에서 실패하는 이유는 과적합 때문이다.
   - 모델의 복잡성을 최소화하고, 아웃오브샘플 검증을 철저히 한다.

3. **분산된 알파 소스 (Diversified Alpha Sources)**
   - 단일 전략이나 단일 시장에 의존하지 않는다.
   - 수천 개의 약한 신호(weak signals)를 결합하여 강한 엣지를 구축한다.

4. **적응형 시스템 (Adaptive Systems)**
   - 시장은 변한다. 시스템도 시장 변화에 적응해야 한다.
   - 고정된 규칙이 아닌 지속적으로 학습하는 모델을 추구한다.

5. **실행의 정밀성 (Execution Precision)**
   - 엣지가 작을수록 실행 비용의 중요성이 커진다.
   - 슬리피지, 수수료, 시장 충격을 최소화하는 실행 알고리즘이 핵심이다.

## 대표 전략 유형

### Quantitative Data-Driven Trading (QIM)

QIM의 전략은 전통적인 퀀트 접근을 넘어 기계학습과 대규모 데이터 분석을 결합한다.

| 구성 요소 | 역할 |
|-----------|------|
| **대규모 데이터 수집** | 가격, 거래량, 펀더멘탈, 대체 데이터 등 광범위한 데이터 소스 |
| **패턴 발견 엔진** | 기계학습 알고리즘을 통한 비선형 패턴 탐색 |
| **과적합 방지 체계** | 교차검증, 정규화, 아웃오브샘플 테스트 |
| **포트폴리오 최적화** | 다수의 약한 신호를 결합한 포트폴리오 구축 |
| **자동 실행** | 알고리즘 기반 완전 자동 매매 |

### 핵심 방법론

- **비선형 관계 탐색**: 전통적 선형 모델이 포착하지 못하는 복잡한 데이터 관계 발견
- **다중 시간 프레임**: 초단기부터 중기까지 다양한 시간 프레임에서 작동하는 신호 결합
- **시장 중립**: 방향성 노출을 최소화하고 알파만 추출하는 구조

## 매매 기법

### 진입 (Entry)

- **조건**: 복수의 독립적 퀀트 신호가 동일 방향을 가리킬 때
- **방식**: 완전 자동화. 인간의 개입 없이 모델이 자동 진입
- **분산**: 수백~수천 개 종목에 걸쳐 동시 진입
- **사이즈**: 각 신호의 강도, 신뢰도, 감쇠 속도에 따라 자동 산정

### 청산 (Exit)

- **신호 기반**: 진입 신호가 소멸되거나 반대 신호가 발생하면 청산
- **시간 기반**: 특정 보유 기간이 경과하면 자동 청산
- **리밸런싱**: 정기적 포트폴리오 리밸런싱을 통한 체계적 청산
- **비용 최적화**: 거래 비용을 고려한 최적 청산 타이밍 계산

### 손절 (Stop-Loss)

- **모델 내장**: 개별 손절선이 아닌 포트폴리오 전체의 리스크 한도로 관리
- **변동성 기반**: 각 종목의 변동성에 비례하여 자동 조정
- **드로다운 한도**: 포트폴리오 전체의 최대 허용 드로다운 설정
- **동적 조정**: 시장 레짐에 따라 리스크 한도를 자동 조정

### 포지션 사이징 (Position Sizing)

- **신호 강도 비례**: 신호가 강할수록 더 큰 포지션
- **변동성 역비례**: 변동성이 높을수록 포지션 축소
- **상관관계 고려**: 포지션 간 상관관계를 고려한 포트폴리오 레벨 사이징
- **용량 제한**: 유동성을 고려한 최대 포지션 크기 제한

### 리스크 관리 (Risk Management)

- **체계적 분산**: 종목, 섹터, 전략, 시간 프레임 전반에 걸친 분산
- **팩터 노출 관리**: 시장, 섹터, 스타일 팩터에 대한 노출을 모니터링하고 중립화
- **테일 리스크 관리**: 극단적 시장 상황에 대한 스트레스 테스트
- **모델 리스크 관리**: 단일 모델에 과도하게 의존하지 않는 앙상블 접근

## 심리와 의사결정

### 핵심 멘탈 모델

- **데이터 신뢰**: 인간의 직관보다 데이터 분석 결과를 신뢰한다.
- **겸손한 자세**: 시장을 완벽히 이해할 수 없다는 전제에서 출발한다.
- **과학적 방법론**: 가설 → 테스트 → 검증 → 수정의 과학적 순환을 따른다.
- **감정 배제**: 시스템이 결정하고, 인간은 시스템을 개선하는 역할에 집중한다.

### Woodriff의 의사결정 규칙

1. 과적합을 의심하라 — 너무 좋아 보이면 대부분 과적합이다
2. 단순한 모델이 복잡한 모델보다 대체로 강건하다
3. 데이터가 충분하지 않으면 거래하지 마라
4. 시장이 변했다면 모델도 적응해야 한다
5. 실행 비용을 무시한 백테스트는 무의미하다

## 대표 사례

### 성공 사례: QIM의 장기 안정적 수익

- **배경**: QIM은 설립 이후 장기간에 걸쳐 안정적인 위험조정 수익률을 달성
- **전략**: 수천 개의 약한 신호를 결합한 시장 중립 포트폴리오 운용
- **핵심 요인**:
  - 과적합 방지에 대한 극도의 경계
  - 지속적인 새로운 데이터 소스 발굴
  - 실행 비용 최소화를 위한 알고리즘 최적화
- **교훈**: 화려한 단기 수익보다 안정적 장기 수익이 복리 효과를 극대화한다

### 실패/교훈 사례: 과적합과의 전쟁

- **배경**: 초기 모델 개발 시 백테스트 성과가 우수했으나 실전에서 부진한 경험
- **원인**: 데이터 마이닝 편향(data mining bias)으로 인한 과적합
- **대응**: 엄격한 아웃오브샘플 검증 프로세스 도입, 모델 복잡성 제한
- **교훈**: "백테스트의 적은 항상 과적합이다. 실전 성과만이 유일한 진실이다."

## 내 시스템과의 연결

### SEPA 공통점

| Woodriff 요소 | OmX 구현 |
|---------------|----------|
| 데이터 기반 패턴 발견 | **Alpha 에이전트** — 정량적 스크리닝 조건 자동 평가 |
| 과적합 방지 | **백테스트 프레임워크** — 워크포워드 분석, 아웃오브샘플 검증 |
| 다수 약한 신호 결합 | **Omega 에이전트** — 복수 에이전트 출력 통합 스코어링 |
| 자동 실행 | **실행 엔진** — 신호 발생 → 자동 알림/반자동 실행 |

### SEPA 차이점

| 항목 | Woodriff/QIM | OmX 시스템 |
|------|-------------|-----------|
| 접근법 | 순수 퀀트, 인간 개입 최소 | 퀀트 + 재량 보조 하이브리드 |
| 모델 복잡성 | 기계학습, 비선형 모델 | 규칙 기반 + 선형 모델 중심 |
| 자산 범위 | 글로벌 다자산 | 한국/미국 주식 중심 |
| 거래 빈도 | 고빈도~중빈도 | 중빈도~저빈도 (스윙~포지션) |
| 포지션 수 | 수백~수천 | 소수 집중 (4~10개) |

### 전략군 (Strategy Family)

- **Quant**: 순수 정량적 접근. 데이터 분석과 기계학습 기반의 체계적 전략

### 에이전트 매핑

| 에이전트 | Woodriff 요소 | 구현 내용 |
|----------|--------------|----------|
| **Alpha** | 패턴 발견 | 가격/거래량 패턴의 통계적 유의성 검증 |
| **Beta** | 다중 신호 결합 | VCP + 추가 패턴 신호 앙상블 |
| **Omega** | 포트폴리오 최적화 | 복수 에이전트 스코어 가중 결합 |
| **백테스트** | 과적합 방지 | 워크포워드, 부트스트래핑, 현실적 비용 반영 |

### 구현 아이디어

1. **과적합 감지 시스템**: 인샘플 vs 아웃오브샘플 성과 괴리를 자동 모니터링
2. **약한 신호 앙상블**: 개별적으로 약한 Alpha/Beta/Gamma 신호를 통계적으로 결합하여 강한 종합 신호 생성
3. **적응형 파라미터**: 시장 레짐에 따라 모델 파라미터를 자동 조정하는 메타 레이어
4. **실행 비용 시뮬레이터**: 슬리피지, 스프레드, 시장 충격을 포함한 현실적 실행 비용 모델링
5. **모델 성과 모니터**: 각 에이전트/모델의 실시간 성과를 추적하고 성과 저하 시 자동 경고
