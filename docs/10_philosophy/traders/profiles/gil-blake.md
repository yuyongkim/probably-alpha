# Gil Blake

## 한 줄 요약

> **"시장의 단기 사이클을 통계적으로 포착하고, 뮤추얼 펀드 간 타이밍 전환으로 일관된 수익을 추구한다."**

## 핵심 투자 철학 (5 Principles)

1. **시장에는 반복 패턴이 존재한다 (Markets Have Recurring Patterns)**
   - 시장의 단기 움직임에는 통계적으로 유의미한 반복 패턴이 있다.
   - 이 패턴을 체계적으로 식별하고 활용하면 일관된 수익이 가능하다.

2. **시스템이 감정을 대체한다 (System Replaces Emotion)**
   - 모든 매매 결정을 사전 정의된 시스템에 따라 실행한다.
   - 인간의 감정적 판단을 배제하고, 검증된 규칙에만 의존한다.

3. **일관성이 수익을 만든다 (Consistency Creates Returns)**
   - 화려한 대박이 아닌 작고 일관된 수익의 반복이 장기 복리를 만든다.
   - 월 승률 95%+는 각 거래의 기대값이 아닌 일관성의 결과이다.

4. **리스크 제어가 수익보다 중요 (Risk Control > Returns)**
   - 12년간 연간 기준 단 1개월만 손실 — 이는 극도의 리스크 제어의 산물.
   - "수익은 리스크 관리의 부산물이다."

5. **단순한 시스템이 복잡한 시스템을 이긴다 (Simple Beats Complex)**
   - 과최적화(overfitting)를 피하기 위해 단순한 규칙을 선호.
   - 파라미터 수가 적을수록 미래에도 작동할 가능성이 높다.

## 대표 전략 유형

### Systematic Mutual Fund Timing

Blake의 전략은 뮤추얼 펀드 간 체계적 전환 타이밍이다:

| 구성 요소 | 역할 |
|-----------|------|
| **사이클 분석** | 시장 단기 사이클의 통계적 패턴 식별 |
| **타이밍 모델** | 주식형 ↔ 채권형/머니마켓 펀드 전환 규칙 |
| **모멘텀 지표** | 단기 모멘텀 방향 확인 |
| **리스크 필터** | 불확실성 높은 환경에서는 현금(머니마켓) 유지 |
| **자동 실행** | 규칙 기반 자동 전환, 감정 개입 배제 |

## 매매 기법

### 진입 (Entry)

- **시스템 신호**: 사전 정의된 모델이 매수 신호를 생성할 때만 진입
- **모멘텀 확인**: 단기 모멘텀이 상승 방향일 때 주식형 펀드로 전환
- **사이클 타이밍**: 시장 단기 사이클의 바닥 근처에서 주식형 진입
- **확인 지연**: 신호 발생 후 1일 확인 기간을 둔 후 실행 (가짜 신호 필터)

### 청산 (Exit)

- **반대 신호**: 시스템이 매도 신호를 생성하면 주식형 → 채권형/머니마켓 전환
- **모멘텀 약화**: 단기 모멘텀이 하락 방향으로 전환되면 청산
- **사이클 정점**: 시장 단기 사이클의 정점 근처에서 방어적 전환
- **시간 기반**: 평균 보유 기간 3~10일. 장기 보유 지양

### 손절 (Stop-Loss)

- **시스템 기반**: 개별 손절선이 아닌 시스템의 반대 신호가 손절 역할
- **최소 손실**: 뮤추얼 펀드 특성상 개별 주식보다 변동성이 낮아 손실 폭이 제한적
- **빠른 전환**: 신호 반전 시 즉시 전환. 지체 없이 실행
- **최대 드로다운**: 월간 기준 드로다운 한도 설정. 초과 시 현금 전환

### 포지션 사이징 (Position Sizing)

- **올인/올아웃**: 시스템 특성상 100% 주식형 또는 100% 머니마켓으로 전환
- **레버리지 미사용**: 뮤추얼 펀드 투자로 레버리지 불필요
- **자금 규모 조정**: 자산 규모 증가에 따른 전략 용량(capacity) 관리
- **복수 펀드**: 여러 펀드 패밀리에 분산하여 실행 리스크 분산

### 리스크 관리 (Risk Management)

- **월 승률 95%+**: 극도로 높은 승률로 드로다운 자체를 방지
- **현금 = 방어**: 불확실할 때는 머니마켓 펀드에서 이자 수익 확보
- **전략 모니터링**: 시스템 성과를 지속 추적, 유의미한 성과 저하 시 재검증
- **과최적화 방지**: 단순한 규칙 유지, 과도한 파라미터 조정 금지

## 심리와 의사결정

### 핵심 멘탈 모델

- **시스템 신뢰**: 시스템을 100% 신뢰하고 따른다. 의심이 드는 순간 시스템이 무용해진다
- **인내**: 시스템이 신호를 줄 때까지 기다린다. 강제로 매매를 만들지 않는다
- **겸손**: "내가 시장을 이기는 것이 아니라, 시스템이 시장의 패턴을 포착하는 것이다"
- **과정 집중**: 개별 거래의 결과가 아닌 시스템의 장기 성과에 집중

### Blake의 심리 규칙

1. 시스템을 따르라. 시스템을 오버라이드하는 순간 시스템은 사라진다
2. 일관성이 천재성보다 중요하다. 매일 같은 규칙을 같은 방식으로 실행하라
3. 드로다운은 시스템의 일부다. 드로다운에 패닉하지 마라
4. 최적화의 유혹을 이겨라. 단순한 규칙이 미래에도 작동한다
5. 자신의 성과를 냉정하게 추적하라. 숫자는 거짓말하지 않는다

## 대표 사례

### 성공 사례: 12년간 연평균 45%, 월 승률 95%+

- **배경**: Blake는 뮤추얼 펀드 타이밍 전략으로 12년간 연평균 45% 수익, 단 1개월만 손실
- **전략**: 시장 단기 사이클의 통계적 패턴을 활용한 주식형 ↔ 머니마켓 전환
- **핵심**: 극도로 높은 일관성. 큰 수익이 아닌 작은 수익의 반복
- **교훈**: 체계적 시스템과 철저한 규율이 인간의 직관을 압도할 수 있다

### 성공 사례: 통계적 엣지 발견

- **배경**: Blake는 시장의 단기 움직임에서 통계적으로 유의미한 패턴을 독자적으로 발견
- **방법**: 수천 일의 시장 데이터를 분석하여 반복 패턴 식별, 백테스트로 검증
- **결과**: 단순하지만 견고한 타이밍 모델 구축
- **교훈**: "시장의 비효율성은 크지 않지만 체계적으로 포착하면 의미 있는 수익이 된다"

### 실패 사례: 전략 용량 한계

- **배경**: 뮤추얼 펀드 타이밍 전략은 자산 규모 증가에 따라 실행이 어려워짐
- **원인**: 뮤추얼 펀드의 빈번한 전환에 대한 제한(short-term trading fees 등)
- **대응**: 여러 펀드 패밀리로 분산, ETF 등 대안 수단 탐색
- **교훈**: 우수한 전략도 용량(capacity) 한계가 있다. 규모가 성과를 잠식할 수 있다

## 내 시스템과의 연결

### SEPA 공통점

Blake의 체계적 타이밍 접근은 OmX 시스템의 전체 아키텍처와 철학적으로 깊이 연결된다:

| Blake 원칙 | OmX 구현 |
|-----------|----------|
| 시스템 기반 매매 | **전체 시스템** — 모든 결정을 알고리즘으로 자동화 |
| 단기 사이클 포착 | **Beta 에이전트** — 단기 모멘텀/사이클 패턴 감지 |
| 모멘텀 방향 | **Alpha 에이전트** — Trend Template 기반 모멘텀 필터 |
| 일관된 실행 | **자동화 파이프라인** — 감정 배제, 규칙 기반 실행 |
| 과최적화 방지 | **백테스트 엔진** — 워크포워드 검증, 최소 파라미터 원칙 |

### SEPA 차이점

| 항목 | Blake 방식 | OmX 시스템 |
|------|-----------|-----------|
| 매매 대상 | 뮤추얼 펀드 (인덱스 단위) | 개별 주식 |
| 진입/청산 | 올인/올아웃 (100% 전환) | 개별 종목별 진입/청산 |
| 보유 기간 | 3~10일 | Stage 2 포지션 (수주~수개월) |
| 포지션 수 | 1개 (펀드 전환) | 다수 종목 동시 보유 |
| 숏 매매 | 없음 (현금 전환만) | 없음 (현물 롱 위주) |

### 전략군 (Strategy Family)

- **Family C (Short-Term Swing)**: Blake의 핵심 전략군. 체계적 단기 타이밍

### 에이전트 매핑

| 에이전트 | Blake 요소 | 구현 내용 |
|----------|-----------|----------|
| **Beta** | 단기 사이클 | 시장 단기 사이클 패턴 자동 감지 |
| **Beta** | 모멘텀 타이밍 | 단기 모멘텀 방향 전환 신호 |
| **Alpha** | 시장 방향 | 시장 전체의 추세 방향 (M factor) |
| **Delta** | 리스크 필터 | 불확실성 지수 기반 현금 전환 신호 |
| **Omega** | 시스템 통합 | 다중 신호 통합 → 최종 진입/대기 결정 |

### 구현 아이디어

1. **시장 단기 사이클 감지**: 과거 N일간의 가격 패턴에서 통계적으로 유의미한 반복 사이클 자동 추출
2. **워크포워드 백테스트**: 과최적화 방지를 위한 롤링 윈도우 백테스트 자동 실행
3. **시스템 성과 모니터링**: 전략의 최근 성과를 실시간 추적, 유의미한 성과 저하 시 자동 경보
4. **올인/올아웃 vs 점진적 진입 비교**: Blake식 전량 전환과 SEPA식 점진적 진입의 성과 비교 분석
5. **월간 승률 추적 대시보드**: 월별 승/패 기록, 연속 승리/패배 추적으로 시스템 건강도 모니터링
