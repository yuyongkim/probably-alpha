# Steve Watson

## 한 줄 요약

> **"좋은 전략도 나쁜 실행 앞에서는 무력하다 — 시장에서의 승리는 아이디어가 아니라 실행의 품질에서 결정된다."**

## 핵심 투자 철학 (5 Principles)

1. **실행이 전략을 이긴다 (Execution Beats Strategy)**
   - 같은 전략이라도 실행 품질에 따라 결과가 크게 달라진다.
   - 주문 타이밍, 슬리피지 관리, 포지션 구축 방식 등 실행의 세부 사항이 누적되어 큰 차이를 만든다.
   - "아이디어는 10%이고, 실행이 90%다."

2. **슬리피지와 마찰 비용 최소화 (Minimizing Friction Costs)**
   - 거래 수수료, 스프레드, 슬리피지, 시장 충격(market impact)은 장기적으로 성과를 크게 잠식한다.
   - 이 비용을 최소화하는 것이 수익을 올리는 것만큼 중요하다.
   - "1%의 슬리피지가 연간 복리 수익을 수 퍼센트포인트 갈아먹는다."

3. **주식 시장에 대한 깊은 이해 (Deep Understanding of Stock Market Microstructure)**
   - 주문 흐름, 시장 조성자의 행동, 유동성의 시간대별 변화를 이해해야 좋은 실행이 가능하다.
   - 호가창(order book)의 움직임에서 단기 수급을 읽는 능력이 실행 품질의 기반이다.
   - 시장 미시구조에 대한 이해가 실행 엣지의 원천이다.

4. **인내와 타이밍 (Patience in Execution)**
   - 좋은 가격에 진입하기 위해 기다리는 인내가 필요하다.
   - 급하게 시장가로 주문하면 불필요한 비용을 지불한다.
   - "좋은 가격에 사는 것이 좋은 종목을 사는 것만큼 중요하다."

5. **리스크 관리는 실행에서 시작된다 (Risk Management Starts at Execution)**
   - 손절 주문의 실행 품질이 리스크 관리의 첫 번째 방어선이다.
   - 갭다운, 유동성 부족 등 실행 리스크를 사전에 고려해야 한다.
   - 실행 불가능한 리스크 관리 계획은 계획이 아니다.

## 대표 전략 유형

### 실행 품질 중심 주식 트레이딩 (Execution-Quality Stock Trading)

Watson의 전략은 실행의 세부 사항을 최적화하여 추가 수익을 창출하는 구조다:

| 구성 요소 | 역할 |
|-----------|------|
| **종목 선별** | 기본적 펀더멘탈/기술적 분석을 통한 후보 선정 |
| **실행 최적화** | 주문 타입, 타이밍, 크기를 최적화하여 마찰 비용 최소화 |
| **유동성 분석** | 종목별 유동성 프로파일을 분석하여 실행 전략 수립 |
| **시장 미시구조 활용** | 호가창, 주문 흐름 분석을 통한 단기 수급 파악 |
| **실행 성과 추적** | VWAP, Implementation Shortfall 등으로 실행 품질 측정 |

### 실행 최적화 기법

1. **지정가 주문 우선**: 시장가 주문 최소화, 지정가로 유리한 가격 확보
2. **시간대 활용**: 개장/마감 시간의 유동성 패턴을 활용한 주문 타이밍
3. **분할 매수/매도**: 대규모 주문을 분할하여 시장 충격 최소화
4. **암묵적 비용 추적**: 스프레드, 슬리피지, 타이밍 비용 등 숨겨진 비용을 정량화
5. **주문 흐름 분석**: 호가창의 불균형에서 단기 방향성을 읽어 실행 타이밍 결정

## 매매 기법

### 진입 (Entry)

- **유동성 윈도우 활용**: 거래량이 풍부한 시간대(개장 30분, 마감 30분)에 진입
- **지정가 매수**: 스프레드 내 유리한 가격에 지정가 주문, 추격 매수 자제
- **분할 진입**: 목표 포지션의 25-50%씩 분할하여 평균 진입가 개선
- **기술적 확인**: 지지/저항, 이동평균선 등 기술적 수준과 연계한 진입 가격 설정

### 청산 (Exit)

- **분할 청산**: 일시에 전량 매도하지 않고, 2-3단계로 분할 실행
- **유동성 확보 후 청산**: 충분한 유동성이 있는 시간대에 청산
- **트레일링**: 추세 지속 시 트레일링 스톱으로 수익 극대화
- **시장 충격 최소화**: 대량 매도 시 TWAP, VWAP 알고리즘 활용

### 손절 (Stop-Loss)

- **실행 가능한 손절**: 갭다운 리스크, 유동성 부족을 고려한 현실적 손절 수준 설정
- **사전 설정 필수**: 진입과 동시에 손절 주문 입력
- **슬리피지 예산**: 손절 실행 시 예상 슬리피지를 손절 계획에 포함
- **유동성 필터**: 일 거래량이 충분하지 않은 종목은 진입 자체를 제한

### 포지션 사이징 (Position Sizing)

- **유동성 기반 사이징**: 일 평균 거래량의 일정 비율 이내로 포지션 제한 (시장 충격 방지)
- **리스크 기반**: 거래당 총 자본의 1-2% 리스크
- **스프레드 고려**: 스프레드가 넓은 종목은 포지션 크기를 축소
- **청산 용이성**: 3일 이내에 전량 청산 가능한 크기로 제한

### 리스크 관리 (Risk Management)

- **실행 리스크 예산**: 슬리피지, 스프레드 등 실행 비용을 리스크 예산에 포함
- **유동성 모니터링**: 보유 종목의 거래량 변화를 추적, 유동성 감소 시 포지션 축소
- **갭 리스크 대비**: 실적 발표, 뉴스 이벤트 전 포지션 축소 또는 헤지
- **일일 실행 리포트**: 모든 주문의 실행 품질을 VWAP 대비로 추적

## 심리와 의사결정

### 핵심 멘탈 모델

- **디테일 중심**: 큰 그림보다 실행의 세부 사항에 주의를 기울인다. "악마는 디테일에 있다."
- **인내심**: 좋은 가격을 기다리는 인내가 장기적으로 큰 차이를 만든다.
- **비용 의식**: 모든 거래에서 숨겨진 비용을 인식하고 최소화하려는 의식을 갖는다.
- **겸손**: 시장은 항상 예상보다 비효율적이거나 효율적일 수 있다. 실행에 대한 과신을 경계한다.

### Watson의 심리 규칙

1. 급할수록 천천히 — 서두르는 주문이 가장 비싼 주문이다
2. 슬리피지 0.5%가 연 수익률 수 퍼센트를 결정할 수 있다
3. 시장가 주문은 마지막 수단이다 — 지정가가 기본이다
4. 실행 성과를 측정하지 않으면 개선할 수 없다
5. 유동성이 부족한 종목은 아무리 매력적이어도 위험하다

## 대표 사례

### 성공 사례: 실행 최적화를 통한 수익 개선

- **배경**: 동일한 전략을 사용하되, 실행 방식을 체계적으로 최적화하여 연간 수익률을 수 퍼센트포인트 개선
- **전략**: 분할 매수, 시간대 최적화, 지정가 주문 활용으로 평균 실행 가격을 VWAP 대비 개선
- **핵심 요인**:
  - 모든 주문의 실행 품질을 VWAP 대비로 추적
  - 실행 데이터를 분석하여 최적 주문 시간대와 주문 크기를 도출
  - 지속적인 실행 프로세스 개선
- **교훈**: 1% 실행 개선이 장기 복리에서 엄청난 차이를 만든다

### 실패/교훈 사례: 유동성 부족의 대가

- **배경**: 펀더멘탈이 매력적이지만 유동성이 부족한 소형주에 과도한 포지션 구축
- **원인**: 유동성 리스크를 과소평가, 시장 충격 비용을 계산에 포함하지 않음
- **대응**: 청산에 수일이 소요되며 불리한 가격에 매도, 슬리피지 손실 발생
- **교훈**: "진입할 때의 유동성이 아니라 청산할 때의 유동성으로 포지션 크기를 결정하라. 위기 시 유동성은 사라진다."

| 유동성 분석 | **Alpha 에이전트** — 거래량 필터, 유동성 기준 |

### SEPA 차이점

| 항목 | Watson 원본 | OmX 시스템 |
|------|-----------|-----------|
| 실행 초점 | 핵심 엣지 (실행 자체가 수익 원천) | 보조 기능 (전략이 수익 원천) |
| 시장 미시구조 | 호가창, 주문 흐름 심층 분석 | 일봉/분봉 수준 분석 |
| 주문 타입 | 다양한 주문 타입 활용 (IOC, FOK 등) | 기본 주문 타입 (지정가, 시장가) |

### 전략군 (Strategy Family)

- **Mixed**: Watson의 핵심 전략군. 주식 시장에서의 실행 품질 중심 트레이딩
- 모든 전략군의 실행 레이어에 적용 가능한 범용적 원칙 제공

### 에이전트 매핑

| 에이전트 | Watson 요소 | 구현 내용 |
|----------|-----------|----------|
| **실행 엔진** | 주문 최적화 | 슬리피지 최소화, 분할 주문, 시간대 최적화 |
| **Alpha** | 유동성 필터 | 거래량 기반 유동성 스크리닝, 유동성 부족 종목 배제 |
| **Delta** | 비용 관리 | 거래 비용(스프레드, 슬리피지)을 리스크 계산에 포함 |
| **Logger** | 실행 성과 추적 | VWAP 대비 실행 품질, 누적 슬리피지 자동 추적 |

### 구현 아이디어

1. **VWAP 실행 엔진**: 대규모 주문을 VWAP 알고리즘으로 분할 실행하여 시장 충격 최소화
2. **유동성 히트맵**: 종목별, 시간대별 유동성을 시각화하여 최적 실행 시간 결정 지원
3. **슬리피지 분석기**: 모든 거래의 슬리피지를 자동 계산하고, 누적 비용을 대시보드에 표시
4. **실행 품질 리포트**: 일/주/월 단위로 실행 품질(VWAP 대비, 스프레드 비용)을 자동 리포트
5. **유동성 기반 사이징**: 종목의 일 평균 거래량에 기반하여 최대 포지션 크기를 자동 제한하는 로직
